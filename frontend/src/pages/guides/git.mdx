### Tagging og versjoner

Tags brukes til √• markere spesifikke versjoner av koden din, spesielt nyttig for releases og n√•r du vil referere til en bestemt versjon senere. Det funker p√• en samme m√•te som en branch, men er konstant til en commit.

**Lag en tag:**

```bash

git tag -a v1.0.0 -m "Release version 1.0.0"  # Annotert tag (anbefalt)
```

**Se tags:**

```bash

git show v1.0.0  # Vis tag info
```

**Push tags:**

```bash
git push origin v1.0.0  # Push spesifikk tag
git push origin --tags  # Push alle tags
```

**Slett tag:**

```bash

git push origin --delete v1.0.0  # Slett remote tag
```

### Fork arbeidsflyt

